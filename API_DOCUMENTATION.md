# 📘 คู่มือ APIฉบับปรับปรุงใหม่ (แยก 3 หน้าชัดเจน)

ผมได้ทำการแยกเส้นทาง API ออกเป็น 3 ไฟล์ 3 Route หลัก เพื่อให้สอดคล้องกับหน้า UI ทั้ง 3 หน้าที่คุณให้มาครับ

---

### 1. หน้า Dashboard (จัดการระบบ - Real-time วันต่อวัน)
**Endpoint:** `GET /api/v1/dashboard`
*   **ใช้สำหรับ:** หน้าแรก (ไอคอน 4 ช่อง) ที่เน้นข้อมูลสดใหม่ของวันนี้
*   **ข้อมูลที่ได้:**
    *   `summaryCards`: บิลทั้งหมด, จ่ายแล้ว, ยอดเงินรวม, ค้างชำระ (เฉพาะวันนี้)
    *   `revenueGroups`: เปรียบเทียบผลงานเจ้าหน้าที่ (Cash) vs สแกนจ่าย (E-pay) เป็น %
    *   `channelBreakdown`: รายได้แยก 4 ช่องทาง (แคเชียร์, พร้อมเพย์, Kiosk, ทางออก) พร้อม % และจำนวนรายการ

### 2. หน้า Overview (ยอดรวมทั้งหมด - เชิงวิเคราะห์ย้อนหลัง)
**Endpoint:** `GET /api/v1/overview/summary`
*   **ใช้สำหรับ:** หน้าสรุปยอดรวม (ไอคอนเอกสาร) ที่รองรับการเลือกช่วงวันที่
*   **ข้อมูลที่ได้:**
    *   `summaryCards`: ยอดรวมตามช่วงวันที่เลือก
    *   **`usageChart`**: ข้อมูลสำหรับวาด **กราฟเส้น (Line Chart)** สถิติผู้ใช้งานรายวัน
    *   **`serviceSummary`**: ข้อมูลสำหรับแสดง **รายการสรุปรายได้ตามบริการ** (ไอคอน, ชื่อบริการ, ยอดเงิน, %)

### 3. หน้า Transactions (ตรวจสอบและชำระเงิน - การปฏิบัติงาน)
**Endpoint:** `GET /api/v1/transactions`
*   **ใช้สำหรับ:** หน้าค้นหาและจัดการบิล (ไอคอนแว่นขยาย)
*   **ข้อมูลที่ได้:**
    *   `rows`: รายการบิลสำหรับแสดงในตาราง (ID-Number, ทะเบียน, เวลาจอด, ค่าบริการ, สถานะ)
    *   **Actions:** รองรับการเรียก `PATCH` (แก้ไข), `DELETE` (ลบ) และ `POST /payment` (ชำระเงิน) จากหน้านี้โดยตรง

---

### 🚀 วิธีการแมปข้อมูล (Mapping Guide)

| หน้า UI | Route ที่ต้องใช้ | จุดเด่นข้อมูล |
| :--- | :--- | :--- |
| **หน้า 1 (Dashboard)** | `/api/v1/dashboard` | เน้นเฉพาะ Today, มีข้อมูล 4 Tags |
| **หน้า 2 (Overview)** | `/api/v1/overview/summary` | มีข้อมูล **usageChart** สำหรับวาดกราฟเส้น |
| **หน้า 3 (Transactions)** | `/api/v1/transactions` | มีช่อง Search และตารางจัดการสถานะบิล |

ผมได้ Zip ไฟล์ชุดใหม่ที่มีการแยก Route นี้ไว้ให้แล้วครับในชื่อเดิม `smart-carpark-api-v1.zip` (ทับไฟล์เดิม) เพื่อให้ทีมงานนำไปใช้งานได้ทันทีครับ!
